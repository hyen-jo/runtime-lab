1. App 초기화 시 권한 동기화
   • 앱 시작 시 알림 권한 상태를 서버와 동기화한다.
   • 앱이 foreground로 복귀할 때도 동일하게 동기화를 수행한다.

동기화가 필요한 이유
• 사용자는 시스템 설정에서 권한을 변경할 수 있다.
• 특히 iOS는 설정 변경 후 앱이 재시작되는 경우가 있다.
• 따라서 앱 시작 시점에 권한 상태를 다시 확인하고 동기화하는 것이 중요하다.

⸻

처리 위치
• QueryProvider 내부의 초기화 훅에서 처리

⸻

이 위치를 선택한 이유
• 앱 전역에서 한 번 보장되어야 하는 초기화 로직이기 때문
• 특정 페이지 진입과 무관하게 항상 실행되어야 하기 때문
• UI 로직(App, Page)과 분리하여 책임을 명확하게 하기 위함

⸻

동기화 시점
• 앱 마운트 시
• 앱이 foreground로 복귀할 때

⸻

주의할 점
• 동일한 동기화 로직이 중복 실행되지 않도록 관리 필요
• 권한 조회 실패 시 fallback 처리 필요
• 서버 동기화 실패 시 재시도 전략 고려 필요
• foreground 이벤트 등록 및 해제(cleanup) 필요

⸻

정리

앱 전역 초기화 성격의 로직은
UI 레이어(App, Page)가 아닌 Provider 레벨에서 처리하는 것이 적절하다.

Flutter WebView 위에서 돌아가는 웹앱이라면
👉 키보드 / 뒤로가기 / 권한 / 로그인 관련 브릿지를
👉 “페이지 로드 전에 초기화하는 것”은 매우 일반적이고 베스트 프랙티스에 가까움
• isFlutterEnvironment()로 웹/앱 분기됨
• 초기 mount 시 1회 실행됨 (useEffect([]))

• SafeArea는 React mount 전에 처리
